# Screenshots Guide

Use this guide to create consistent, public-safe screenshots for the GitHub README and portfolio.

## Before capturing

1. Run the application with **synthetic sample data only**.
2. Remove uploaded files that contain private or real user data.
3. Check that no private filename, local username, email address, phone number, account ID, or confidential text is visible.
4. Collapse browser bookmarks, developer tools, notifications, and unrelated windows.
5. Use a clean browser window at approximately 90% zoom. Adjust to 80–100% if necessary so tables remain readable.
6. Use a viewport around 1440 × 900 or larger for consistent wide-layout captures.

## Capture quality

- Use PNG for crisp interface text.
- Avoid heavy cropping that removes context.
- Keep Chinese and English labels readable.
- Use consistent browser zoom and window width across captures.
- Do not edit scores or evidence in a way that misrepresents the running application.

## Final privacy check

All 7 screenshots must use the bundled synthetic sample data only. Treat screenshots as publishable data and inspect every image at full size before committing it. Do not include private data, real customer information, personal filenames, credentials, email addresses, phone numbers, or confidential text. If uncertain, remove the screenshot and recapture it safely.
